When something goes wrong

发布不成功时，
按这个顺序检查。

01打开网址显示 404

依次确认以下三项，通常就能找到原因：

  1. 仓库根目录存在小写文件名 index.html，不是 index.htm 或 Index.html。
  2. Settings → Pages 中选择的是 main 和 / (root)。
  3. 仓库 Actions 页中的 pages build and deployment 已显示绿色对勾。
02网页能打开，但没有样式

GitHub Pages 的文件名区分大小写。检查 HTML 中引用的名称是否与仓库文件完全一致。

<link rel="stylesheet" href="./styles.css">
03修改后网页没有更新

先在仓库中确认新提交已经出现，再查看 Actions 是否部署完成。完成后等待一两分钟并强制刷新浏览器。

Windows：Ctrl + F5　macOS：Command + Shift + R

04看不到 Settings 或 Pages

你需要拥有该仓库的管理员权限。使用免费个人账号时，建议先用公开仓库完成本课程。
