# GitHub Learning Roadmap 设计说明

## 目标

制作一个面向中文初学者的单页学习网站，让用户按顺序掌握 Git、GitHub 和网站发布。第一版既是可评审的视觉原型，也可以直接部署到 GitHub Pages。

## 视觉方向

- 采用高端科技与智能汽车官网常见的产品叙事方式，而非复制具体品牌。
- 页面通过暖白、纯白、浅灰、深石墨与黑色区块形成节奏。
- 使用大字号标题、严谨网格、精细分隔线和克制的钴蓝色强调。
- 以开放式版面、横向路线和单一重点学习面板为主，避免卡片堆叠、炫光和过度玻璃效果。
- 动效用于路线推进、内容揭示与按钮反馈，并支持减少动态效果的系统偏好。

## 页面结构

1. 极简导航：学习路线、实践项目、资源中心、继续学习。
2. 首屏：核心价值主张、开始学习按钮、总体进度，以及抽象 Git 分支视觉。
3. 八阶段路线：Git 基础、本地仓库、GitHub 仓库、分支协作、GitHub Pages、自定义域名、自动化部署、Next.js + Vercel。
4. 当前课程：展示阶段目标、四个可勾选任务、命令示例和预计时间。
5. 实践成果：个人主页、作品集网站、Next.js 项目。
6. 页尾行动区：引导用户从第一阶段开始。

## 交互

- 点击路线阶段切换当前课程内容。
- 勾选任务后更新阶段进度和总体进度。
- 学习状态保存到当前浏览器的 `localStorage`。
- “继续学习”定位到第一个未完成阶段。
- 代码命令支持一键复制并显示反馈。

## 技术方案

- 原生 HTML、CSS 和 JavaScript，不安装第三方依赖。
- 学习内容与进度逻辑分离，便于后续扩展。
- 响应式适配桌面、平板和手机。
- 使用 Node 内置测试验证进度计算与阶段选择逻辑。

## 安全与发布

- 不包含 API Key、账号凭据或用户敏感信息。
- `localStorage` 只保存任务完成状态。
- 可通过 GitHub Pages 静态发布；后续接入登录或数据库时再增加鉴权、RLS 和权限检查。
